## Hyperparameter optimization

To determine the optimal configuration of the ridge2f model, a Bayesian optimization procedure is implemented in order to select the hyperparameters that minimize predictive uncertainty while ensuring numerical stability of the model. The optimization focuses on the main parameters governing the dynamic structure and the regularization of the model, namely the number of lags, the number of hidden nodes in the random feature layer, and the two ridge penalty parameters.

The selection criterion is based on the Winkler score computed from the prediction intervals generated by the model. The Winkler score evaluates the quality of interval forecasts by jointly penalizing excessive width and insufficient coverage. This criterion is particularly suitable in the context of economic scenario generation, where both accuracy and reliable uncertainty quantification are required.

To preserve the temporal dependence of the data, the optimization relies on a rolling time-series cross-validation scheme. For each candidate set of parameters, the model is estimated on a training window and evaluated on the subsequent forecast horizon. This procedure is repeated over multiple time slices, and the median Winkler score across all validation windows is used as the objective function to be minimized.

The exploration of the parameter space is performed using Bayesian optimization. At each iteration, the algorithm proposes a new set of parameters based on previous evaluations, balancing exploration of the parameter space and exploitation of the best solutions found so far. This approach allows an efficient search without requiring an exhaustive grid evaluation.

After convergence of the optimization procedure, the best parameter vector obtained is

$$
(\text{lags}, \text{nb\_hidden}, \log_{10} \lambda_1, \log_{10} \lambda_2) = (4.63, 19.57, 0.31, 0.85),
$$

which corresponds approximately to

$$
\text{lags} \approx 5, \quad \text{nb\_hidden} \approx 20, \quad \lambda_1 \approx 2.05, \quad \lambda_2 \approx 7.1.
$$

This configuration yields the lowest Winkler score among all tested parameter combinations, indicating that it provides the best compromise between predictive accuracy and reliability of the simulated prediction intervals.

The optimization results show that the model benefits from a nonlinear representation with a sufficiently large number of hidden nodes, while moderate regularization is required to maintain stability. The selected parameter values therefore correspond to a specification that balances flexibility and robustness, which is essential for generating realistic economic scenarios.

The final set of parameters obtained through Bayesian optimization is retained for the estimation of the model and for the subsequent generation of economic scenarios.

## Out-of-sample validation

To assess the predictive performance of the proposed model, we conduct an out-of-sample validation using a hold-out sample corresponding to the last 15% of the observations. The model is estimated on the training set, and forecasts are generated over the testing period. 

Forecast accuracy is evaluated using three complementary metrics. First, the Mean Absolute Error (MAE) and the Root Mean Squared Error (RMSE) are used to assess point forecast accuracy. Second, the Winkler score is used to evaluate the quality of the prediction intervals at the 95% confidence level. The Winkler score jointly penalizes interval width and lack of coverage, making it particularly suitable for probabilistic scenario generators.

Table 1 reports the results for the five variables included in the generator: CAC40 returns, industrial production growth, inflation, short-term interest rate, and long-term interest rate.

| Inflation | 0.19 | 0.24 | 1.31 |
| Short rate | 1.05 | 1.38 | 4.52 |
| Long rate | 0.98 | 1.14 | 3.29 |
